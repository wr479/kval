# Корочки.есть — запись на курсы ДПО

**Дата актуализации:** 12 мая 2026 г.

## Требования

- **Node.js** версии **22.5.0** или новее (нужен встроенный модуль [`node:sqlite`](https://nodejs.org/api/sqlite.html)).
- npm (идёт вместе с Node).

## Запуск

### Docker (рекомендуется)

Нужны [Docker](https://docs.docker.com/get-docker/) и Docker Compose. В каталоге проекта:

```bash
docker compose up --build
```

Сайт: **http://localhost:3000**. База SQLite хранится в томе Docker `korochki_data` (данные не пропадут при пересборке образа).

Другой порт на хосте, например 8080:

```powershell
$env:PORT=8080; docker compose up --build
```

Для продакшена задайте свой `SESSION_SECRET` в `.env` или в окружении.

### Локально (Node.js)

В каталоге проекта:

```bash
npm install
npm start
```

Откройте в браузере: **http://localhost:3000**

Порт по умолчанию — **3000**. Другой порт: переменная окружения `PORT`, например `set PORT=8080` (Windows) или `PORT=8080 npm start`.

### Ошибка «EADDRINUSE» / «address already in use»

Порт **3000** уже используется другим приложением или зависшим процессом Node. Либо завершите процесс:

```powershell
netstat -ano | findstr :3000
taskkill /PID <номер_из_последней_колонки> /F
```

Либо запустите сервер на свободном порту, например **3001**:

```powershell
$env:PORT=3001; npm start
```

## База данных

Файл SQLite создаётся автоматически: **`data/korochki.db`** (каталог `data/` добавляется при первом запуске).

При запуске через Docker файл лежит внутри тома **`korochki_data`** (имя проекта в compose — `korochki-portal`). Скопировать на хост: `docker cp korochki-portal-web-1:/app/data/korochki.db .` (при запущенном контейнере).

### ER-диаграмма

Таблицы `users` и `applications` создаются скриптом при старте сервера. Чтобы построить ER-диаграмму средствами СУБД (п. 1.2 методички):

1. Установите [DB Browser for SQLite](https://sqlitebrowser.org/) или другой клиент SQLite.
2. Откройте файл `data/korochki.db`.
3. Используйте встроенный инструмент схемы / ER (зависит от программы).

## Структура проекта

- `server.js` — Express, API, сессии.
- `db.js` — подключение SQLite и создание таблиц, учётная запись администратора.
- `validation.js` — проверка полей на сервере.
- `public/index.html`, `public/app.js` — интерфейс, запросы через `fetch` без перезагрузки страницы.

## Примечание

При старте Node может вывести предупреждение, что SQLite в составе Node находится в статусе experimental — на работу приложения это не влияет.
